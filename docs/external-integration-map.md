# External Integration & Dependency Surface

This document enumerates the external services and adapters this Mastodon monorepo talks to — backing stores, object storage, ActivityPub federation peers, SSO/auth providers, translation, push, CAPTCHA, email, observability, and other outbound HTTP dependencies — recording for each whether it is **required** or **optional/feature-flagged** (with the controlling flag or configuration) and the degradation behavior only where the codebase's own fallback or error-handling logic evidences it (otherwise `—`). PostgreSQL `Source: config/database.yml:L2` and Redis `Source: config/initializers/sidekiq.rb:L7` are the required backing stores and outbound ActivityPub federation is intrinsic to a federated server `Source: app/lib/activitypub/activity.rb:L3`; the remaining integrations below are optional and individually gated — most by a dedicated environment flag, with a few controlled by other mechanisms such as the `fasp` experimental-feature list `Source: config/mastodon.yml:L3` or a permission-mounted dashboard `Source: config/routes.rb:L24` — as each row's controlling-flag column documents.
