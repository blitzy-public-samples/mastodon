# External Integration & Dependency Surface

This document enumerates every external integration point of this Mastodon monorepo — backing stores, object storage, federation peers, auth providers, translation, push, and observability — reverse-engineered from the initializers and configuration files under `config/`. Each row records whether the integration is **required** or **optional/feature-flagged** (with the controlling flag), its primary file(s), and what degrades if it is unavailable. Per the evidence-only rule, a degradation claim is made **only where the codebase's own fallback or error-handling logic demonstrates it**; otherwise the cell is left as `—`.

| --- | --- | --- | --- |
| PostgreSQL | **Required** (no flag) | `config/database.yml` `Source: config/database.yml:L2` | Required — `adapter: postgresql` is the only configured adapter and the sole datastore; no fallback is defined. `Source: config/database.yml:L2` |
| Redis | **Required** (no flag) | `config/initializers/sidekiq.rb` `Source: config/initializers/sidekiq.rb:L7` | Required — both the Sidekiq server and client bind their connection to Redis, so queues and background processing cannot operate without it. `Source: config/initializers/sidekiq.rb:L7` `Source: config/initializers/sidekiq.rb:L103` |
