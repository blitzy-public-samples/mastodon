# Feature & Component Map

This document inventories the major functional areas of this Mastodon monorepo — what each area is, where it lives, and how critical it is — reverse-engineered from the route definitions and the primary directory groupings under `app/`. **Criticality here is purely technical: it reflects each area's position in the dependency structure (how close it sits to the core request path and how many other areas depend on it), not business priority, roadmap status, or organizational intent.** Areas are listed most-critical first; the five most-critical rows additionally list the files a new engineer should read first.
