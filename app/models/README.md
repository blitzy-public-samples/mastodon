# app/models

The ActiveRecord **domain & persistence layer** — the application's core data entities. Database-backed models inherit from the abstract base `ApplicationRecord` (`class ApplicationRecord < ActiveRecord::Base` with `primary_abstract_class`); a minority of non-persistence/helper models in this directory subclass a different base or none at all (e.g., `AccountFilter`, `Context < ActiveModelSerializers::Model`, `UserSettings`). `Source: app/models/application_record.rb:3-4`, `Source: app/models/account_filter.rb:3`, `Source: app/models/context.rb:3`, `Source: app/models/user_settings.rb:3`

## Conventions

- **`concerns/` mixin pattern** — Shared model behavior is extracted into modules under `app/models/concerns/` (e.g., `Cacheable`, `Paginable`), including an `account/` sub-namespace (e.g., `Account::Associations`), and mixed into models with `include` (e.g., `Status` includes `Cacheable`, `Discard::Model`, `Paginable`, and several `Status::*` concerns). _Why:_ it lets cross-cutting behavior be reused across many models without duplication. `Source: app/models/concerns/cacheable.rb:3`, `Source: app/models/concerns/paginable.rb:3`, `Source: app/models/concerns/account/associations.rb:3`, `Source: app/models/status.rb:35-46`
- **Top-of-file schema annotations** — Database-backed ActiveRecord models commonly open with a `# == Schema Information` block listing their columns and indexes, auto-generated by AnnotateRB and governed by the repo-root `.annotaterb.yml` (`:position: before`); non-persistence/helper models (e.g., `ApplicationRecord` itself) carry no such block. _Why:_ an engineer can see the table shape inline without opening the schema or the database. `Source: app/models/account.rb:3`, `Source: .annotaterb.yml:2`, `Source: app/models/application_record.rb:3`
- **Local vs. remote `Account`** — `Account` defines `scope :remote, -> { where.not(domain: nil) }` and `scope :local, -> { where(domain: nil) }`, with matching `#local?` / `#remote?` predicates. _Why:_ a `nil` `domain` is a user on this instance, while a non-`nil` `domain` is a cached representation of a foreign actor — the mechanism by which federation stores remote data. `Source: app/models/account.rb:147-148`, `Source: app/models/account.rb:200-206`
