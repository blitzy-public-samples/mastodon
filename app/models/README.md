## Conventions

- **`concerns/` mixin pattern** — Shared model behavior is extracted into modules under `app/models/concerns/` (including an `account/` sub-namespace) and mixed into models with `include` (e.g., `Status` includes `Cacheable`, `Discard::Model`, `Paginable`, and several `Status::*` concerns). _Why:_ it lets cross-cutting behavior be reused across many models without duplication. `Source: app/models/status.rb:35-46`
- **Top-of-file schema annotations** — Each model opens with a `# == Schema Information` block listing its columns and indexes, auto-generated by AnnotateRB and governed by the repo-root `.annotaterb.yml` (`:position: before`). _Why:_ an engineer can see the table shape inline without opening the schema or the database. `Source: app/models/account.rb:3`, `Source: .annotaterb.yml:2`
- **Local vs. remote `Account`** — `Account` defines `scope :remote, -> { where.not(domain: nil) }` and `scope :local, -> { where(domain: nil) }`, with matching `#local?` / `#remote?` predicates. _Why:_ a `nil` `domain` is a user on this instance, while a non-`nil` `domain` is a cached representation of a foreign actor — the mechanism by which federation stores remote data. `Source: app/models/account.rb:147-148`, `Source: app/models/account.rb:200-206`
